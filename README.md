REST-сервис для управления электронными кошельками. Поддерживает операции пополнения и снятия средств с гарантией атомарности транзакций при высокой конкурентной нагрузке.

## Архитектурные решения

### Обработка конкурентных запросов

Ключевой вызов проекта — обеспечение корректной работы при нагрузке до 1000 RPS на один кошелек без потерь транзакций и ошибок 5xx.

**Проблема классических подходов:**

- Пессимистичные блокировки (`SELECT FOR UPDATE`) при высокой конкуренции приводят к длительным ожиданиям и таймаутам.
- Оптимистичные блокировки (`@Version`) генерируют `OptimisticLockingFailureException`, требующие сложной логики повторов.
- Оба подхода увеличивают риск 5xx ошибок под нагрузкой.

**Примененное решение:**

Атомарное обновление баланса на уровне СУБД одним SQL-запросом:

```sql
UPDATE wallet 
SET balance = balance + :amount, 
    version = version + 1 
WHERE id = :id 
  AND (balance + :amount) >= 0
